# Schedulers Artifact

This repository contains the data/model download helpers, vision experiments, and transformer experiments used by the scheduler artifact.

## Layout

- `download_datasets.py`: downloads and prepares `20news`, `cifar100`, `e2e`, and `tiny-imagenet` under `data/`.
- `download_models.py`: downloads GPT-2 small and DistilBERT checkpoints under `model/`.
- `nn_models.py`: shared vision model definitions.
- `vision_scheduler/`: CIFAR-100 and Tiny-ImageNet experiments.
- `transformer_scheduler/`: 20News and E2E experiments.
- `data/`, `model/`, and `*/save/` are runtime outputs and are ignored by git.

## Environment

Python 3.10 is recommended.

```bash
cd artifact
python -m venv .venv
. .venv/bin/activate
pip install -r requirements.txt
```

Install the PyTorch build that matches your CUDA driver if the generic `torch` wheel is not appropriate for the machine.

E2E generation metrics also require Java because METEOR and PTBTokenizer are run from bundled `.jar` files.

## Prepare Data and Models

Run from the artifact root:

```bash
python download_datasets.py
python download_models.py
```

This creates:

```text
data/20news
data/cifar100
data/e2e
data/tiny-imagenet
model/gpt2
model/distilbert-base-uncased
```

Both download scripts recreate their target directories.

## Run Experiments

Each run script pins `CUDA_VISIBLE_DEVICES=0` and uses `PYTHON="${PYTHON:-python}"`.

Vision:

```bash
cd vision_scheduler
sh run_cifar100_main.sh
sh run_cifar100_ablation.sh
sh run_tiny_imagenet_main.sh
sh run_tiny_imagenet_ablation.sh
```

Transformer:

```bash
cd transformer_scheduler
sh run_20news_main.sh
sh run_20news_ablation.sh
sh run_e2e_main.sh
sh run_e2e_ablation.sh
```

Experiment directories are generated by each scheduler's `src/arguments.py`. Use `--eval_mode main` for final-round evaluation and `--eval_mode ablation` for tracked-client evaluation. Log names end with the selected mode, either `main` or `ablation`.

## Result Scripts

Vision:

```bash
cd vision_scheduler
python cifar100_main_results.py
python cifar100_ablation_results.py
python tiny_imagenet_main_results.py
python tiny_imagenet_ablation_results.py
```

Transformer:

```bash
cd transformer_scheduler
python 20news_main_results.py
python 20news_ablation_results.py
python e2e_main_results.py
python e2e_ablation_results.py
```

The result scripts call the same `generate_log_name()` functions as the run scripts, so save-directory names stay synchronized with the experiment arguments.
The `*_main_results.py` scripts summarize final-round logs. The `*_ablation_results.py` scripts save tracking plots as PNG files under each scheduler's `figures/` directory.

## Ablation Evaluation

With `--eval_mode ablation`, the schedulers track representative clients `0`, `1`, and `2`. When any tracked client participates in a round, the code evaluates:

- `server.txt`: the server model before aggregation at that round.
- `proxy_<id>.txt`: the tracked client's proxy model.
- `proxy_train_<id>.txt`: the tracked client's trained proxy model.
- `proxy_finetune_<id>.txt`: the tracked client's finetuned proxy model.

With `--eval_mode main`, the schedulers keep the main experiment behavior and evaluate only the final round, including server, selected proxy models, finetuned proxy models, and collusion averages.

## Runtime Outputs

Do not commit generated data, models, logs, IPC files, or result directories. They are ignored by `.gitignore`:

```text
data/
model/
*/save/
**/_ipc/
__pycache__/
*.pyc
```
